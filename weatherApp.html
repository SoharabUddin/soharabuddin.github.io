Weather App

A comprehensive Android application for real-time weather updates.

Project Overview

The Weather App is an Android application developed using Kotlin that provides real-time weather information for any specified city. It leverages the WeatherAPI to fetch and display current weather data, including temperature, wind speed, humidity, and weather conditions.

Key Features

  • Real-time Weather Data
  • Error Handling
  • LiveData and ViewModel
  • Retrofit
  • Gson
  • Responsive UI

Technologies Used

  • Kotlin
  • Android Architecture Components
  • Retrofit
  • Gson
  • WeatherAPI

How It Works

The Weather App fetches real-time weather data from the WeatherAPI based on the user's input city. The main components involved in this process are:

  • User Input: The user enters the city name in the app.
  • API Request: The app sends a request to the WeatherAPI using Retrofit, including the city name and API key.
  • Data Processing: The API response is processed using Gson to convert JSON data into Kotlin data classes.
  • LiveData Update: The processed data is posted to LiveData objects, which the UI observes for changes.
  • UI Update: The UI components display the updated weather data in a user-friendly format.

Components

MainViewModel.kt

The MainViewModel class is responsible for managing the data for the Weather App. It uses LiveData to hold the current weather data and handles the API requests to fetch this data.

  • getWeatherData(city: String): Initiates the API request to fetch weather data for the specified city.
  • onResponse(call: Call<CurrentWeatherResponse>, response: Response<CurrentWeatherResponse>): Handles the API response and updates LiveData objects accordingly.
  • onFailure(call: Call<CurrentWeatherResponse>, t: Throwable): Handles errors during the API request and updates the error LiveData object.

ServiceBuilder.kt

The ServiceBuilder object provides the Retrofit instance configured to communicate with the WeatherAPI.

  • createService(service: Class<T>): T: Creates and returns a Retrofit service instance for the specified service interface.

WeaterAPI.kt

The WeaterAPI interface defines the API endpoints for the WeatherAPI service.

  • getCurrentWeather(query: String, apiKey: String, language: String): Call<CurrentWeatherResponse>: Defines the API endpoint to fetch current weather data.

CurrentWeatherResponse.kt

The CurrentWeatherResponse data class represents the structure of the weather data returned by the WeatherAPI.

  • Location: Nested data class representing the location details.
  • Current: Nested data class representing the current weather details.

ErrorResponse.kt

The ErrorResponse data class represents the structure of error messages returned by the WeatherAPI.

  • ErrorDetails: Nested data class representing the details of the error.
